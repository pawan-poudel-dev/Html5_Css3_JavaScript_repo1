1. HOME PAGE

Comparison with Beeswrap.com

Visual Comparison

Beeswrap (Reference)

EcoMart Home Page

Reference Website:

https://www.beeswrap.com/

Why Referenced: Leading eco-friendly brand with excellent banner design, product showcasing and sustainability messaging. Strong focus on visual storyTelling and customer testimonials.

Similarities

  • Both websites focuses on eco-friendly/sustainable products.
  • Both have top navigation or products showcase features .
  • "Shop Now" CTA (a visual queue (a clickable button or link)) buttons both have same designs.
  • Both include customer reviews/brand story.

Differences

  • Bee's wrap is a fully functional in cart, checkout, backend and uses databases
  • Our website is a static website(HTML/CSS/ Basics of Javascript only) no real cart logic.
  • Bee's wrap has professional UI/UX designs, filters , and accessibility.
  • Navigation: EcoMart has search bar in header; Beeswrap has top navigation only

Improvements Made

Implemented product image with hover slider.
Clear top categories section and link them to their own category.
Well structured and commented HTML
Better color contrast for WCAG compliance(WCAG compliance means making a website easy to use for everyone, including people with disabilities)

2. PRODUCTS PAGE

Comparison with Amazon.com

Visual Comparison

Amazon (Reference)

EcoMart Products Page

Reference Website:

https://www.amazon.com/

Why Referenced: Industry-leading e-commerce platform with optimized product discovery, filtering, sorting, and efficient catalog organization. Gold standard for e-commerce UX.

Similarities

  • Both websites shows products image, rating stars, discount and price.
  • Both have Add to Cart / Buy Now / Wishlist buttons which helps to buy products
  • Both focus on helping users decide before buying
  • Both useing clear call-to-action buttons(means that buttons that guide users clearly to the next action.)

Differences

  • Amazon uses a fully dynamic ecommerce system (real cart, checkout, user account, backend and other advance tools and technologies).
  • Our site is static (HTML/CSS/JS only), no real purchase flow which doesnot have any backend features. Its just a sample of a website.
  • Amazon has detailed descriptions, reviews, delivery & return informartion which helps users to easily filter and read all the information.
  • Our website has limited products and details.

Improvements Made

Added images cycling on product hover for multiple views.
Implemented category carousels for easier navigation.
Better discount display in red color with percentage savings.
We have implemented well-structured and readable HTML code.
Added sustainability badges/labels for eco-products.
Good foundation for adding backend later.

5. LOGIN PAGE

Custom Authentication Design

Visual Comparison

Standard Login (Reference)

EcoMart Login Page

Reference Website:

https://dribble.com/

Why Referenced: Secure and user-friendly authentication page with background imagery, form validation, "Remember Me" checkbox, Google Authentication option, and password security.

Similarities
